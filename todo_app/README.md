# Daily To-Do

A fully offline Flutter to-do app with 5 tabs: Everyday Tasks, Weekly
Dashboard, Today's Goals, Monthly Dashboard, and Normal Tasks. All data is
stored on-device with Hive — nothing is ever sent over the network.

## What's included

This package contains the complete **`lib/`** source code and **`pubspec.yaml`**
for the app — every screen, model, provider, and piece of logic described
below. It does **not** include the `android/`, `ios/`, etc. platform
folders, because those are auto-generated by the Flutter tool itself and
it's more reliable to let `flutter create` generate them fresh on your
machine than to hand-write them.

## How the 5 tabs work

1. **Everyday Tasks** — create once, tick every day. A task's completion
   isn't stored on the task itself; it's stored per-day in a separate
   `DailyCompletion` record (`taskId + date`). That's what makes the
   "reset at midnight" requirement free: a new day has no record yet, so
   the box is unticked automatically — no background job needed. History
   is never deleted. Drag the handle on the right to reorder; tap the
   pencil/trash to edit or delete.
2. **Weekly Dashboard** — reads the same `DailyCompletion` records for
   Monday–Saturday (Sunday is intentionally skipped) of the selected week,
   with arrows to move between weeks. Header shows the ISO week number and
   date range.
3. **Today's Goals** — a `TodayGoal` is tagged with the date it was created
   for (`forDate`). The screen only shows goals where `forDate == today`,
   so yesterday's goals silently drop out of view while staying in Hive
   for history.
4. **Monthly Dashboard** — for each everyday task, counts how many days
   (from the 1st of the month through today, or the full month if you're
   looking at a past month) have a completed record, and shows a progress
   bar + percentage, plus an overall total.
5. **Normal Tasks** — one-time tasks with title, optional description,
   status, created date, and completed date. Swipe left to delete (with
   an Undo snackbar), drag to reorder.

Settings (gear icon on the Everyday tab) has a dark/light mode toggle and a
"reset all data" option.

## Architecture

- **State/DI:** Riverpod (`flutter_riverpod`). Hive boxes are exposed as
  `Provider`s; widgets rebuild live via `ValueListenableBuilder` on
  `box.listenable()`, which is the standard, minimal-boilerplate way to
  wire Hive into Flutter without duplicating state in memory.
- **Storage:** Hive, with hand-written `TypeAdapter`s (no `build_runner`
  step needed — one less thing that can break on a fresh machine).
- **Routing:** `go_router` with a `StatefulShellRoute.indexedStack` so each
  tab keeps its own navigation state.
- **Models:** `RecurringTask`, `DailyCompletion`, `TodayGoal`, `NormalTask`
  — see `lib/models/`.

## Setup — turning this into an installable APK

You'll need the Flutter SDK installed (https://docs.flutter.dev/get-started/install).

```bash
# 1. Scaffold a fresh Flutter project (this generates android/, ios/, etc.)
flutter create daily_todo
cd daily_todo

# 2. Replace the generated lib/ and pubspec.yaml with the ones from this package
rm -rf lib
cp -r /path/to/this/package/lib .
cp /path/to/this/package/pubspec.yaml .

# 3. Get packages
flutter pub get

# 4. Run on a connected device/emulator
flutter run

# 5. Build a release APK to install on your phone
flutter build apk --release
# The APK will be at build/app/outputs/flutter-apk/app-release.apk
```

Copy that APK to your phone (or run `flutter install` with the phone
connected via USB with developer mode/USB debugging on) and open it to
install — you may need to allow "install from unknown sources" the first
time.

## Notes / things you may want to tweak

- App name & icon: after `flutter create`, edit `android/app/src/main/AndroidManifest.xml`
  (the `android:label`) and swap the icons in `android/app/src/main/res/mipmap-*`
  (an easy way to do both is the `flutter_launcher_icons` package).
- Export/Import/Backup were intentionally left out of this first version to
  keep things simple — the Settings screen currently only has a theme
  toggle and a full reset. Happy to add JSON export/import if useful.
